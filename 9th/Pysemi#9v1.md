# Pythonセミナー第9回 v1モデルの設計と実行結果

作成日：2026年7月20日

対象ノートブック：[Pysemi#9v1.ipynb](./Pysemi%239v1.ipynb)

提出用予測：[submissionv1.csv](./submissionv1.csv)

用語調査：[Pysemi#9search.md](./Pysemi%239search.md)

## 予測処理の流れ

今回のモデルは、配布された訓練CSVのSMILESと `gap_eV` だけを使って学習する。

処理は次の順序で進む。

1. 訓練CSVとテストCSVを読み込み、列名と欠損値を検査する。

2. RDKitでSMILESをCanonical SMILESへ変換し、同じ分子の重複を確認する。

3. RDKitの2次元分子記述子、補助グラフ特徴量、Morganカウントフィンガープリントを生成する。

4. 一意な訓練分子を、シード8で学習用80%と検証用20%へ分ける。

5. XGBoostを学習し、検証用データのMAEを計算する。

6. 検証で採用した木の本数を使い、一意な全訓練分子からモデルを学習し直す。

7. テストデータ4,000件を予測し、`smiles,gap_eV` の2列を持つ `submissionv1.csv` を保存する。

この順序にした理由は、テストデータの正解値を使わずにモデルの性能と学習回数を決めたあと、利用できる訓練データを最終モデルへすべて渡すためである。

## レギュレーションとの対応

| レギュレーション | v1での対応 |
|---|---|
| MAEで評価 | 検証データの平均絶対誤差を計算した |
| データの追加は禁止 | 配布された二つのCSV以外の分子データや物性値を使っていない |
| 提出可能なモデルは最大3個 | v1ではXGBoostを一つだけ使用した |
| 0からモデルを学習 | XGBoostを配布訓練データから新しく学習した |
| LLMとNNPの組込みは禁止 | どちらも予測処理へ組み込んでいない |
| モデルの予測値を使用 | `gap_eV` には最終XGBoostの出力をそのまま使用した |
| シード値は自由、8を推奨 | データ分割とXGBoostの乱数シードを8に固定した |
| 提出形式は `smiles,gap_eV` | 列名、行数、SMILESの順序、有限値を保存前後に検査した |

RDKitは事前学習済み予測モデルではなく、SMILESから分子構造を読み取り、数値特徴量へ変換するために使っている。

## Canonical SMILESと重複処理

同じ分子でも、原子をたどる順序によって異なるSMILESを記述できる。

そこでRDKitのCanonical SMILESへそろえたうえで、重複分子と訓練データからテストデータへの重なりを調べた。

実行結果は次のとおりだった。

| 確認項目 | 結果 |
|---|---:|
| 元の訓練行数 | 15,000 |
| Canonical SMILESで重複した行数 | 2 |
| 一意な訓練分子数 | 14,998 |
| 訓練データとテストデータの重複分子数 | 0 |

重複した二つの行は、同じCanonical SMILESに同じ `gap_eV` が付いていた。

重複を残したまま無作為分割すると、同じ分子が学習用と検証用へ分かれ、検証MAEが実力以上によくなる可能性がある。

そのため、検証と最終学習にはCanonical SMILESが一意な14,998分子を使用した。

ただし、提出CSVではテストCSVの元のSMILESと行順を変更していない。

## 特徴量の構成

特徴量は合計2,277列になった。

| 特徴量群 | 列数 | 表す情報 |
|---|---:|---|
| RDKitの2次元分子記述子 | 214 | 分子量、極性、環数、官能基数、電子的環境など |
| 補助グラフ特徴量 | 15 | 元素数、原子数、結合種、芳香族原子数、共役結合数など |
| Morganカウントフィンガープリント | 2,048 | 各原子を中心とする局所部分構造の出現回数 |
| 合計 | 2,277 | 分子全体の性質と局所構造を連結した表現 |

### 2次元分子記述子を選んだ理由

分子記述子は、分子量や芳香環数のように分子全体の性質を少数の数値で表す。

HOMO-LUMOギャップは元素組成、結合種、芳香族性、共役の広がりと関係するため、これらを表す記述子は予測の手掛かりになる。

配布CSVには3次元座標がないため、v1ではSMILESだけから決定的に再計算できる2次元記述子へ限定した。

RDKitで3次元構造を生成する方法もあるが、その構造はQM9のDFT最適化構造そのものではなく、構造生成と最適化の条件も増える。

v1で2次元へ限定すると、特徴量の由来と再現条件を説明しやすい。

### 補助グラフ特徴量を加えた理由

RDKitの記述子だけに任せず、C、N、O、Fの個数と、単結合、二重結合、三重結合、芳香族結合、共役結合の個数を明示的に加えた。

QM9はC、H、N、O、Fからなるため、元素数は分子の組成を直接表す。

結合種と共役の数は、飽和構造とπ電子系の違いを直接数える。

同じ情報を一部のRDKit記述子も含むが、木モデルは有用でなければその列を分岐へ使わないため、v1では解釈しやすさを優先して残した。

### Morganカウント型を選んだ理由

分子記述子だけでは、特定の原子配列や官能基の位置関係が同じ集約値へ埋もれる場合がある。

Morganフィンガープリントは、各原子の近傍にある局所構造をハッシュ化して固定長のベクトルへ記録する。

v1では半径2、2,048列、キラリティ情報ありの設定に固定した。

ビット型は部分構造の有無だけを0と1で表すが、カウント型は同じ部分構造が何回現れるかも残す。

QM9のような小分子でも、同じ結合環境が複数回現れる場合があるため、v1ではカウント型を採用した。

2,048列へ圧縮すると異なる構造が同じ列へ入るハッシュ衝突は起こりうるが、列数を増やしすぎると計算量も増える。

2,048列は、衝突と計算量の均衡を取るための初期設定である。

## 欠損値の扱い

RDKitの記述子には、分子によって計算不能な値が生じる可能性がある。

ノートブックは無限大を欠損値へ置き換え、各列の中央値で補完できるようにしている。

検証時の中央値は学習用80%だけから計算し、検証用20%へ同じ値を適用する。

検証データを含めて中央値を計算すると、正解値は使わなくても検証側の特徴量分布が学習処理へ混ざるためである。

今回の実行では、訓練特徴量とテスト特徴量のどちらにも欠損値は発生しなかった。

それでも補完処理を残したのは、RDKitの版や入力分子が変わった場合にも、原因を確認したうえで同じ流れを実行できるようにするためである。

## XGBoostを選んだ理由

今回の特徴量は、連続値の分子記述子と、多数のゼロを含むMorganカウント列が混在する。

XGBoostは決定木を順番に追加し、前の木が残した誤差を後の木で修正する勾配ブースティングモデルである。

決定木は特徴量の標準化を必要とせず、連続値とカウント値の非線形な組合せを扱える。

訓練データ約15,000件に対して2,277列を使う今回の規模でも、ヒストグラム方式なら現実的な時間で学習できる。

主な設定は次のとおりである。

| 設定 | 値 | 意図 |
|---|---:|---|
| 目的関数 | 二乗誤差回帰 | 連続値の `gap_eV` を予測する |
| 評価指標 | MAE | スライドの指定に合わせる |
| 学習率 | 0.03 | 一本ずつの修正量を小さくする |
| 木の深さ | 7 | 複雑な相互作用を表しつつ深すぎる木を避ける |
| 最小子ノード重み | 2.0 | 少数データだけに合わせた分岐を抑える |
| 行の使用率 | 0.90 | 各木で一部の行を変え、過学習を抑える |
| 列の使用率 | 0.75 | 各木で一部の特徴量を変え、特定列への依存を抑える |
| L2正則化 | 2.0 | 葉の予測値が極端になることを抑える |
| 木の上限 | 4,000 | 小さい学習率で十分な回数を確保する |
| 早期停止の待機回数 | 150 | 検証MAEが150回改善しなければ停止する |
| 乱数シード | 8 | スライドの推奨値へ固定する |

v1では4,000本目まで検証MAEが更新されたため、早期停止は発動せず上限へ到達した。

これは「4,000本が最適だと確定した」という意味ではなく、今回調べた範囲では4,000本が最良だったことを意味する。

計算時間との均衡を取り、v1では4,000本を最終モデルへ採用した。

## 検証方法と結果

Canonical SMILESで重複を除いた14,998分子を、シード8で次のように分割した。

| 分割 | 分子数 |
|---|---:|
| 学習用 | 11,998 |
| 検証用 | 3,000 |

検証用3,000分子に対する結果は次のとおりだった。

| 指標 | 結果 |
|---|---:|
| MAE | 0.201413 eV |
| 採用した木の本数 | 4,000 |

このMAEは、正解が非公開のテストデータに対する得点ではない。

訓練データから無作為に分けた検証部分だけに対する値である。

無作為分割は同じQM9由来の分布に対する目安になるが、骨格が大きく異なる分子への外挿性能までは保証しない。

## 特徴量重要度から読めること

検証モデルで重要度が高かった特徴量には、二重結合数、sp3炭素割合、共役結合数、芳香族原子数、芳香族結合数などが含まれた。

HOMO-LUMOギャップがπ共役や飽和度の影響を受けることと整合する結果である。

上位にはケトン、アミド、アルデヒドなどの官能基記述子と、Morganフィンガープリントの一部も入った。

ただし、XGBoostの特徴量重要度は予測で使われた分岐の寄与を表す指標であり、化学的な因果関係を証明するものではない。

Morgan列はハッシュ化された位置なので、列番号だけでは対応する部分構造を直接解釈できない。

## 最終モデルと提出CSV

検証後、中央値補完を一意な全訓練データ14,998件から計算し直した。

次に、同じ設定と4,000本の木を使うXGBoostを全訓練データから新しく学習した。

最終モデルがテストデータ4,000件へ出した予測だけを `gap_eV` 列へ保存した。

保存結果は次のとおりである。

| 確認項目 | 結果 |
|---|---:|
| 行数 | 4,000 |
| 列 | `smiles,gap_eV` |
| 予測値の最小値 | 3.624192 eV |
| 予測値の最大値 | 9.656164 eV |
| 非数または無限大 | 0件 |
| テストCSVとのSMILES順の不一致 | 0件 |

保存後に `submissionv1.csv` を読み直し、列名、行数、SMILESの順序、予測値が保存前と一致することも確認した。

## uv環境へ追加した依存関係

次のライブラリを `uv add` でプロジェクトへ追加した。

- `pandas`：CSVと表形式データを扱う。

- `numpy`：特徴量行列を数値配列として扱う。

- `rdkit`：SMILESの解析、分子記述子、Morganフィンガープリントを計算する。

- `scikit-learn`：データ分割、中央値補完、MAE計算を行う。

- `xgboost`：回帰モデルを学習する。

- `matplotlib` と `seaborn`：検証結果と特徴量重要度を描画する。

- `tqdm`：特徴量生成の進行状況を表示する。

- `nbformat` と `nbclient`：ノートブックの検証と自動実行に使う。

実行時の主要な版は、Python 3.11.15、RDKit 2026.03.4、XGBoost 3.2.0、scikit-learn 1.9.0、pandas 3.0.3だった。

## 再実行方法

プロジェクトディレクトリへ移動し、uv環境から次のコマンドを実行する。

```bash
uv run jupyter execute 'Pysemi#9v1.ipynb' --inplace --timeout=-1
```

ノートブックを対話的に開く場合は、このプロジェクトの `.venv` にあるカーネルを選ぶ。

ノートブックを最後まで実行すると、同じディレクトリの `submissionv1.csv` が上書きされる。

## v1の限界

v1は一つの無作為分割だけで設定を評価しているため、分割を変えたときのMAEの変動は測っていない。

XGBoostは4,000本の上限でも改善中だったため、木の本数と学習率を十分に探索し切っていない。

特徴量はSMILESから作れる2次元情報へ限定しており、QM9が持つDFT最適化済み3次元構造は使っていない。

これらはレギュレーション違反ではないが、次の版で精度と検証の安定性を改善できる余地である。
