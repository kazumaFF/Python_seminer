# Pythonセミナー第9回 v2モデルの設計と実行結果

更新日：2026年7月21日

対象ノートブック：[Pysemi#9v2.ipynb](./Pysemi%239v2.ipynb)

提出用予測：[submissionv2.csv](./submissionv2.csv)

用語調査：[Pysemi#9search.md](./Pysemi%239search.md)

## v1からの変更点

v2は、v1の特徴量とXGBoost設定を維持し、検証方法だけを3-fold交差検証へ変更した。

| 項目 | v1 | v2 |
|---|---|---|
| 特徴量 | 2,277列 | 変更なし |
| XGBoostの設定 | 学習率0.03、最大4,000本 | 変更なし |
| 検証方法 | 80%対20%の単一分割 | 3-fold交差検証 |
| 検証用モデルの学習回数 | 1回 | 3回 |
| 学習率の探索 | なし | なし |
| 最終モデルの木数 | 単一分割の最良木数4,000本 | fold別最良木数の中央値3,999本 |

以前のv2に含まれていた5種類の学習率探索は削除した。

これにより、交差検証の学習回数は最大15回から3回へ減る。

## 予測処理の流れ

処理は次の順序で進む。

1. 訓練CSVとテストCSVを読み込み、列名と欠損値を検査する。

2. RDKitでSMILESをCanonical SMILESへ変換し、同じ分子の重複を確認する。

3. RDKitの2次元分子記述子、補助グラフ特徴量、Morganカウントフィンガープリントを生成する。

4. 一意な訓練分子を、シード8で3個のfoldへ分ける。

5. 各foldを一度ずつ検証用にしてXGBoostを学習し、out-of-fold予測を保存する。

6. 全分子のout-of-fold予測からMAEを計算し、fold別MAEと最良木数を記録する。

7. fold別最良木数の中央値を使い、一意な全訓練分子からモデルを学習し直す。

8. テストデータ4,000件を予測し、`smiles,gap_eV` の2列を持つ `submissionv2.csv` を保存する。

## レギュレーションとの対応

| レギュレーション | v2での対応 |
|---|---|
| MAEで評価 | out-of-fold予測の平均絶対誤差を計算する |
| データの追加は禁止 | 配布された二つのCSV以外の分子データや物性値を使わない |
| 提出可能なモデルは最大3個 | 最終予測にはXGBoostを一つだけ使用する |
| 0からモデルを学習 | XGBoostを配布訓練データから新しく学習する |
| LLMとNNPの組込みは禁止 | どちらも予測処理へ組み込まない |
| モデルの予測値を使用 | 最終XGBoostの出力を `gap_eV` へ保存する |
| シード値は自由、8を推奨 | KFoldとXGBoostの乱数シードを8に固定する |
| 提出形式は `smiles,gap_eV` | 列名、行数、SMILESの順序、有限値を保存前後に検査する |

RDKitは予測済みの物性値を与えるモデルではなく、SMILESを数値特徴量へ変換するために使う。

## Canonical SMILESと重複処理

同じ分子でも、原子をたどる順序によって異なるSMILESを記述できる。

そこでRDKitのCanonical SMILESへそろえたうえで、重複分子と訓練データからテストデータへの重なりを調べる。

v2の実行で確認した件数は次のとおりだった。

| 確認項目 | 結果 |
|---|---:|
| 元の訓練行数 | 15,000 |
| Canonical SMILESで重複した行数 | 2 |
| 一意な訓練分子数 | 14,998 |
| 訓練データとテストデータの重複分子数 | 0 |

重複した二つの行には、同じCanonical SMILESと同じ `gap_eV` が入っていた。

重複を残すと、同じ分子が異なるfoldへ入り、交差検証のMAEが実力以上によくなる可能性がある。

そのため、交差検証と最終学習にはCanonical SMILESが一意な14,998分子を使用する。

提出CSVでは、テストCSVの元のSMILESと行順を維持する。

## 特徴量の構成

特徴量はv1と同じ2,277列である。

| 特徴量群 | 列数 | 表す情報 |
|---|---:|---|
| RDKitの2次元分子記述子 | 214 | 分子量、極性、環数、官能基数、電子的環境など |
| 補助グラフ特徴量 | 15 | 元素数、原子数、結合種、芳香族原子数、共役結合数など |
| Morganカウントフィンガープリント | 2,048 | 各原子を中心とする局所部分構造の出現回数 |
| 合計 | 2,277 | 分子全体の性質と局所構造を連結した表現 |

RDKitの2次元分子記述子は、分子量や芳香環数など、分子全体の性質を数値で表す。

補助グラフ特徴量は、C、N、O、Fの個数と、結合種、芳香族性、共役の数を明示的に加える。

Morganカウントフィンガープリントは、各原子の近傍にある局所構造と出現回数を2,048列へ記録する。

半径2、キラリティ情報あり、カウント型という設定もv1から変更しない。

配布CSVには3次元座標がないため、SMILESだけから決定的に再計算できる特徴量へ限定する。

## 欠損値の扱い

RDKitの記述子には、分子によって計算不能な値が生じる可能性がある。

ノートブックは無限大を欠損値へ置き換え、各列の中央値で補完する。

交差検証では、各foldの学習用特徴量だけから中央値を計算し、同じ値をそのfoldの検証用特徴量へ適用する。

全訓練特徴量から補完値を計算すると、検証側の特徴量分布が学習処理へ混ざるためである。

最終学習では、一意な全訓練分子から中央値を計算し直す。

今回の実行では、訓練特徴量とテスト特徴量のどちらにも欠損値はなかった。

| 特徴量行列 | 形状 | 欠損値数 |
|---|---:|---:|
| 訓練データ | 14,998行、2,277列 | 0 |
| テストデータ | 4,000行、2,277列 | 0 |

## XGBoostの設定

特徴量と学習条件を同時に変えると、検証方法を変更した影響を切り分けられない。

そこで、XGBoostの設定はv1と同じ値に固定する。

| 設定 | 値 |
|---|---:|
| 目的関数 | 二乗誤差回帰 |
| 評価指標 | MAE |
| 学習率 | 0.03 |
| 木の深さ | 7 |
| 最小子ノード重み | 2.0 |
| 行の使用率 | 0.90 |
| 列の使用率 | 0.75 |
| L1正則化 | 0.0 |
| L2正則化 | 2.0 |
| 木の上限 | 4,000 |
| 早期停止の待機回数 | 150 |
| 乱数シード | 8 |

学習率の候補比較と最大木数の拡張は行わない。

## 3-fold交差検証

KFoldは14,998分子をシード8でシャッフルし、ほぼ同じ大きさの3個のfoldへ分ける。

各回では2個のfoldで学習し、残りの1個でMAEを計算する。

この処理を3回行うと、すべての分子に学習時未使用のモデルによるout-of-fold予測が一つずつ付く。

実行結果は次のとおりだった。

| fold | 学習分子数 | 検証分子数 | MAE | 最良木数 |
|---:|---:|---:|---:|---:|
| 1 | 9,998 | 5,000 | 0.203529 eV | 3,999 |
| 2 | 9,999 | 4,999 | 0.204877 eV | 3,999 |
| 3 | 9,999 | 4,999 | 0.199715 eV | 4,000 |

| 集計指標 | 結果 |
|---|---:|
| 全14,998分子のout-of-fold MAE | 0.202707 eV |
| fold別MAEの平均 | 0.202707 eV |
| fold別MAEの標準偏差 | 0.002678 eV |
| fold別最良木数の中央値 | 3,999本 |

fold別MAEは0.199715 eVから0.204877 eVまでの範囲に収まり、標準偏差は0.002678 eVだった。

この結果から、シード8で作った3個のfoldの範囲では、分割によるMAEの変動は大きくなかったと判断した。

一方、3個のfoldすべてで最良木数が3,999本または4,000本になった。

最良値が探索上限の近くにあるため、「4,000本付近が十分な木数である」とまでは確認できていない。

最終モデルの木数には、3個の最良木数の中央値である3,999本を使った。

v1の単一分割MAEは0.201413 eVだったため、v2のout-of-fold MAEは0.001294 eV大きい。

ただし、v1とv2では検証対象と各回の学習分子数が異なるため、この差だけからモデルの精度が低下したとは判断できない。

## 依存関係

- `pandas`：CSVと表形式データを扱う。

- `numpy`：特徴量行列を数値配列として扱う。

- `rdkit`：SMILESの解析、分子記述子、Morganフィンガープリントを計算する。

- `scikit-learn`：KFold、中央値補完、MAE計算を行う。

- `xgboost`：回帰モデルを学習する。

- `matplotlib` と `seaborn`：交差検証結果と特徴量重要度を描画する。

- `tqdm`：特徴量生成の進行状況を表示する。

- `nbformat` と `nbclient`：ノートブックの検証と自動実行に使う。

実行時の主要な版は、Python 3.11.15、pandas 3.0.3、scikit-learn 1.9.0、RDKit 2026.03.4、XGBoost 3.2.0だった。

## v2の限界

3-fold交差検証は単一分割より多くの分子を検証へ回せるが、検証モデルの学習はv1の1回から3回へ増える。

シード8による一組のfoldだけを使うため、分割依存性を完全には除けない。

無作為KFoldは同じQM9由来の分布に対する性能の目安であり、骨格が大きく異なる分子への外挿性能までは保証しない。

特徴量はSMILESから作れる2次元情報へ限定しており、QM9が持つDFT最適化済み3次元構造は使っていない。
